Debian / Linux CLI

Install hatsu-fmm from the APT repository

This path is for the packaged FMM CLI. Add the Pages-backed repository, trust the published signing key, then install the binary package with apt.

APT setup 
curl -fsSL https://yoshyhyrro.github.io/hatsu-yakitori/public.asc \
  | gpg --dearmor \
  | sudo tee /usr/share/keyrings/hatsu-yakitori-archive-keyring.gpg >/dev/null

echo "deb [arch=amd64 signed-by=/usr/share/keyrings/hatsu-yakitori-archive-keyring.gpg] https://yoshyhyrro.github.io/hatsu-yakitori stable main" \
  | sudo tee /etc/apt/sources.list.d/hatsu-yakitori.list >/dev/null

sudo apt update
sudo apt install hatsu-fmm

CHICKEN Scheme / source checkout

Use the egg metadata and full module set

This path keeps you in the Scheme toolchain. It is the right fit when you want extensions such as machine_constants, golay_frontier, kak_decomposition, kak_quiver_safety, or the FMM modules directly from source.

Source checkout install 
git clone https://github.com/Yoshyhyrro/hatsu-yakitori.git
cd hatsu-yakitori
chicken-install srfi-1 srfi-69 srfi-95 srfi-4 srfi-133 records
chicken-install -n

Choose the channel

Debian package vs CHICKEN source flow

APT / .deb

Best when you want a clean Linux install of the standalone hatsu-fmm CLI with signed metadata and a predictable apt story.

CHICKEN egg / checkout

Best when you want the wider module surface, Scheme-level experimentation, or to build directly from the repository.

The Pages branch serves both signed APT metadata and a human-facing landing page. The public key lives at public.asc.
